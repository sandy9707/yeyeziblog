|Post View:

过程

青龙面板

docker

添加国内镜像

下载镜像并运行, 注意先复制docker run -dit \再复制后面.

nas

目录规则和原来似乎有一些变化, 我感觉有一个/ql/data/就够了…

原来的教程中的/ql/应该是已经失效了.

为了使用代理方便我设置网络为host, 这样本地代理可以直接用.

faker3(京东)

创建拉取任务

创建任务, 名字随便, 时间随便, 用完就可以删, 我随便设的0 15 10 15 1 1.

依赖安装

免代码安装依赖

点击青龙面板的依赖管理——>新建依赖——>选择NodeJs、自动拆分选择是、复制以下的依赖填到名称里——>点击确定，等待安装完成。

脚本自动(推荐)^[7]^

如果有青龙日志分析 && 自动补全依赖脚本的话, 可以在Configuration Files里面加一句代码解决.

如果有单容器 二叉树修复脚本依赖文件脚本的话, 可以在Configuration Files里面加一句代码解决.

手动

别的可以自动安装, 但是requests需要手动.

考虑到nas有进不去命令行的情况, 可以继续创建任务.

pip install requests

COOKIE获取

最好是用wskey, 可以达到半永久效果.

IOS抓包可以用stream.

按引用[5]的步骤, 找到GET https://im-x.jd.com开头的链接，点进去往下滑，找到cookie里的pin和wskey，但是我只找到了wskey和pin_hash, 万幸经测试pt_pin和pin是等效的.
